# 首页工具扩展路线图
基于现有首页工具实现情况，以下列出需新增/增强的功能点及推荐实施顺序，方便逐一落地。

## P0 · 立即执行
| 顺位 | 功能 | 现状 | 备注与依赖 |
| --- | --- | --- | --- |
| 1 | **图片加水印** | 未实现 | 直接复用 `image-compress` 中的 Canvas 压缩流程，新增水印绘制层（文字/图片），纯前端可落地。 |
| 2 | **Markdown 转 HTML** | Memo 详情已有 Markdown→HTML 渲染 | 抽取 `renderMarkdown` 能力做成独立工具页，提供输入/实时预览/复制 HTML，注意 XSS 过滤。 |

> **执行建议**：两项均可复用既有前端能力，无需后端支持，可在媒体工具页快速补足。完成后更新首页分类展示。

## P1 · 中期规划（纯前端或轻依赖）
| 顺位 | 功能 | 现状 | 备注与依赖 |
| --- | --- | --- | --- |
| 3 | **格式转换（限定图片/音频等轻量场景）** | 未实现 | 明确范围：先做图片格式互转（JPG/PNG/WebP），可借助 Canvas 导出；后续如需文档转换则另立后端。 |
| 4 | **倒计时** | 未实现 | 纯前端 DateDiff + 本地存储即可，可做纪念日/活动倒计时。 |
| 5 | **单位换算** | 未实现 | 维护本地映射表（长度/重量/温度等），前端输入即计算。 |
| 6 | **简单待办** | 未实现（备忘录偏重且需登录） | 提供免登录 ToDo（localStorage）或在 memo 基础上加一个轻量视图。 |

> **执行建议**：按表内顺位逐项实现。每完成一项即可接入首页 `categories`，并在 README/更新日志同步说明。

## P2 · 需后端/第三方支撑
| 顺位 | 功能 | 现状 | 关键依赖 |
| --- | --- | --- | --- |
| 7 | **视频转音频** | 未实现 | 需后端/云函数运行 ffmpeg，处理大文件上传与回传。 |
| 8 | **帧提取** | 未实现 | 同上，依赖 ffmpeg 批量抽帧；注意存储与下载链路。 |
| 9 | **PDF 合并/拆分** | 未实现 | 可选 pdf-lib（H5）但体积&性能堪忧，推荐后端服务。 |
| 10 | **截图转文字（OCR）** | 未实现 | 需 OCR API（腾讯/阿里/自建），涉及成本与隐私合规。 |
| 11 | **汇率换算** | 未实现 | 依赖实时汇率 API，需做 Key 管理或服务端代理。 |

> **执行建议**：P2 功能在启动前需补充后端评估（API 选型、带宽/存储成本、合法域名配置）。可先完成技术可行性调研，再排期开发。

## 实施说明
1. **遵循顺序**：按表格从 P0→P1→P2 顺次推进，单项完成后再进入下一项，确保资源集中。  
2. **复用能力**：优先复用 `image-compress`、memo Markdown 渲染等现有组件，减少重复造轮子。  
3. **交付节奏**：每个功能完成后需：
   - 新增/更新对应 `subPackages/tools/*` 页面。  
   - 在 `pages/index/index.vue` 注册入口并更新 `enabledToolsCount`。  
   - 在 README 或更新日志记录功能上线。  
4. **后端协同**（仅 P2）：在实现前完成 API 设计和资源评估，必要时搭建测试环境与小程序合法域名。
