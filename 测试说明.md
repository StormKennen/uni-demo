# 首页自动登录和快捷服务登录验证测试说明

## 🎯 实现功能

### 1. 首页自动登录检查
- ✅ 页面加载时自动执行 `autoLogin()` 检查
- ✅ 在浏览器控制台可以看到登录检查结果日志

### 2. 快捷服务登录验证
以下服务需要登录才能访问：

**快捷功能区域：**
- 税务服务 (`/pages/calculator/tax-calculator/index`)
- 银行开户 (`/pages/pagesServices/business/company-account-open/index`)
- 商品商城 (`/pages/mall/mall`)
- 订单管理 (`/pages/mine/order/order`)

**热门服务区域：**
- 强制公积金服务 (`/pages/pagesServices/business/mandatory-provident-fund/home`)
- 个人税务服务 (`/pages/pagesPersonalTax/home`)
- 薪俸税服务 (`/pages/pagesSalaryTax/home/index`)

### 3. 无需登录的服务
- 食谱 (`/pages/pagesServices/recipe/index`)
- AI咨询 (`/pages/introduce/introduce`)
- Token测试 (`/pages/test-token`)

## 🧪 测试步骤

### 测试1：未登录状态下点击需要登录的服务
1. 打开首页 `http://localhost:5173/`
2. 确保当前未登录（可以先清除浏览器存储）
3. 点击"税务服务"、"银行开户"、"商品商城"或"订单管理"
4. **预期结果**：显示"请先登录"提示，1秒后跳转到登录页面

### 测试2：已登录状态下点击服务
1. 先通过Token测试页面设置有效Token
2. 点击需要登录的服务
3. **预期结果**：直接跳转到对应服务页面

### 测试3：Token过期状态下点击服务
1. 通过Token测试页面设置过期Token
2. 点击需要登录的服务
3. **预期结果**：
   - 如果refresh token有效：自动刷新token后跳转
   - 如果refresh token也过期：提示登录并跳转到登录页面

### 测试4：无需登录的服务
1. 点击"食谱"、"AI咨询"等服务
2. **预期结果**：直接跳转，无需登录验证

## 🔍 调试信息

### 浏览器控制台日志
- `首页自动登录检查结果:` - 显示自动登录的结果
- `登录检查失败:` - 显示登录检查过程中的错误

### Token测试页面
- 访问 `http://localhost:5173/#/pages/test-token` 
- 可以模拟设置有效/过期Token
- 可以测试自动登录逻辑

## 📝 注意事项

1. **登录页面路径**：`/pages/mine/login/login`
2. **Toast提示时间**：1秒后跳转
3. **错误处理**：登录检查失败时也会跳转到登录页面
4. **性能优化**：使用本地token过期时间判断，减少不必要的网络请求

## 🚀 核心优势

1. **用户体验**：无感知的登录状态检查
2. **安全性**：确保敏感服务需要登录才能访问
3. **智能化**：自动刷新过期token
4. **容错性**：多层fallback机制
5. **可配置**：易于添加/移除需要登录的服务
